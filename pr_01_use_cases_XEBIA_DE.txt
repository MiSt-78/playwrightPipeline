Priority 1 (critical)

1. Title: Main Banner Links Work
- Priority: Priority 1 (critical)
- User Story: "As a visitor, I want to click main banner links so that I reach the expected destinations."
- Clarification: Main banner links are the prominent featured links displayed in the hero/banner section of the homepage. These include contextual CTAs (calls-to-action) and "Mehr Entdecken" (More Discover) links that highlight key service offerings like AI-driven Software Engineering, GenAI Platforms, customer story links, and solution pages. These are temporary/seasonal banners that showcase current initiatives and solutions.
- Acceptance Criteria:
  - Given the homepage is loaded, when I click each main banner link, then the browser navigates to the link's destination.
  - Given a banner link is clicked, when navigation completes, then the destination page displays the expected heading or URL fragment.
  - Given all main banner links are iterated, when each is validated, then there are no broken links (HTTP 4xx/5xx) among them.

2. Title: Top Navigation Links

  - Given each top nav item is tested, when completed, then no top navigation item returns an error page.

3. Title: Homepage Search

5. Title: Cookie Consent Banner
- Priority: Priority 1 (critical)
- User Story: "As a visitor, I want to control cookie consent so that the site respects my privacy choices."
- Clarification: The cookie consent banner is a modal or overlay that appears on first visit to inform users about cookie usage and request consent. It typically offers options to Accept, Decline, or Manage cookie preferences. The choice should be stored and persist across sessions so the banner does not re-appear for returning visitors with saved preferences.
- Acceptance Criteria:
  - Given the homepage is loaded for the first time, then a cookie consent banner is visible.
  - Given I accept cookies, when I confirm, then the banner is hidden and a consent state is stored (banner not shown on subsequent load).
  - Given I reject or manage cookies, when I select that option, then the site records the choice and the banner behavior updates accordingly.

25. Title: Accessibility Features
- Priority: Priority 1 (critical)
- User Story: "As a user with accessibility needs, I want the site to be accessible so that I can navigate and consume content comfortably."
- Clarification: Accessibility testing verifies that the site meets WCAG 2.1 AA standards. This includes keyboard navigation (Tab key), heading structure, alt text for images, color contrast, form labels, focus indicators, and screen reader compatibility. Xebia publishes an Accessibility Statement (/de/accessibility-statement/) detailing their commitment.
- Acceptance Criteria:
  - Given the site is accessed via keyboard only, when I use Tab key to navigate, then all interactive elements are reachable and usable.
  - Given images are on a page, when inspected, then all decorative and informational images have appropriate alt text.
  - Given the site is reviewed for contrast, then text and UI elements meet minimum WCAG AA contrast ratios (4.5:1 for normal text).
  - Given the site is used with a screen reader, then headings, lists, and landmarks are properly announced.
